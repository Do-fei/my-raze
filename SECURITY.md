# 安全政策

## 报告安全漏洞

如果你发现了安全漏洞，请**不要**通过公开的 GitHub Issues 报告。请通过以下方式私密联系我们：

- 通过 GitHub 的 Security Advisory 功能提交

我们会在 48 小时内确认收到报告，并在 7 个工作日内提供修复计划。

## 安全设计

### API Key 管理

My Raze 采用 **BYOK（Bring Your Own Key）** 模式，用户的 API Key 存储在服务端数据库中，不会暴露在前端代码或日志中。所有涉及 API Key 的请求均通过服务端代理转发。

### 数据隔离

所有用户数据（女友配置、对话记录、自拍照片）均与用户 ID 绑定，通过 `protectedProcedure` 确保只有认证用户才能访问自己的数据。

### 认证机制

项目使用 Manus OAuth 进行身份验证，会话通过 JWT 令牌管理，令牌存储在 HttpOnly Cookie 中，防止 XSS 攻击。
