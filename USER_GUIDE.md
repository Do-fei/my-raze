# AI Girlfriend 使用说明

## 项目简介

AI Girlfriend 是一个基于 Clawra 技术的虚拟女友聊天应用，支持智能对话和场景化自拍照片生成。

### 核心功能

1. **智能对话**：基于 LLM API（如 XAI Grok）实现自然流畅的对话
2. **场景化自拍**：根据对话内容自动生成相应场景的自拍照片
3. **个性化配置**：自定义女友的性格、外貌和兴趣爱好
4. **对话历史**：保存聊天记录，支持历史消息查看
5. **自拍画廊**：展示所有生成的自拍照片

## 快速开始

### 1. 首次登录

访问应用后，点击"立即开始"进行登录。

### 2. 创建女友配置

首次登录后，需要创建女友配置：

1. **上传参考照片**：选择一张清晰的正面照片（建议分辨率 512x512 以上）
2. **填写基本信息**：
   - **名字**：给你的虚拟女友起个名字（例如：小雨）
   - **性格特征**：详细描述性格（例如：温柔体贴，善解人意，喜欢撒娇）
   - **外貌描述**：描述外貌特征（例如：长发飘逸，皮肤白皙，身材苗条）
   - **兴趣爱好**（可选）：描述兴趣（例如：喜欢看电影、听音乐、旅行）

3. 点击"创建女友"完成配置

### 3. 配置 API Keys

在设置页面配置必要的 API Keys：

#### fal.ai API Key（必填）

用于生成自拍照片。

1. 访问 [fal.ai/dashboard/keys](https://fal.ai/dashboard/keys)
2. 注册账号并创建 API Key
3. 将 API Key 复制到设置页面

**成本**：约 $0.022/张图片

#### LLM API Key（可选）

用于智能对话。推荐使用 XAI Grok 4.1 Fast。

1. 访问 [console.x.ai](https://console.x.ai/)
2. 注册账号并创建 API Key
3. 将 API Key 复制到设置页面
4. 配置 API URL：`https://api.x.ai/v1/chat/completions`
5. 配置模型名称：`grok-2-1212`

**成本**：约 $0.05-0.10/天（100 轮对话）

> **提示**：如果不配置 LLM API Key，将使用内置的 LLM API（可能有使用限制）

## 使用指南

### 开始聊天

1. 在聊天界面输入消息
2. AI 女友会根据你设置的性格特征进行回复
3. 对话会自动保存，下次登录可继续

### 生成自拍

AI 女友会根据对话内容自动判断何时生成自拍。触发关键词包括：

- "照片"、"自拍"
- "看看"、"发张"、"发个"
- "拍一张"、"拍个"
- "穿"、"在哪"、"在干嘛"
- "现在"、"在做什么"

**示例对话**：

```
你：你现在在哪呀？
AI：我在咖啡厅呢，给你发张照片吧~
[自动生成咖啡厅自拍]

你：今天穿什么衣服？
AI：穿了件红色连衣裙，好看吗？
[自动生成镜子自拍]
```

### 自拍模式

应用会自动选择合适的自拍模式：

#### Mirror Selfie（镜子自拍）

**适用场景**：
- 服装展示（outfit, wearing, clothes, dress）
- 全身照（full-body）
- 镜子自拍（mirror）

**特点**：通过镜子拍摄，适合展示穿搭和整体造型

#### Direct Selfie（直接自拍）

**适用场景**：
- 地点拍摄（cafe, restaurant, beach, park）
- 特写照片（close-up, portrait, face）
- 表情展示（smile, eyes）

**特点**：手机自拍，适合特写和地点拍摄

### 查看自拍画廊

1. 点击聊天界面右上角的图片图标
2. 浏览所有生成的自拍照片
3. 可以下载或删除照片

## 技术架构

### 前端技术栈

- **框架**：React 19 + TypeScript
- **样式**：Tailwind CSS 4 + shadcn/ui
- **路由**：Wouter
- **状态管理**：tRPC + React Query

### 后端技术栈

- **框架**：Express 4 + tRPC 11
- **数据库**：MySQL（通过 Drizzle ORM）
- **存储**：S3（用于图片存储）
- **认证**：Manus OAuth

### 核心技术

1. **Clawra 提示词工程**：
   - 自动检测对话场景
   - 智能选择自拍模式（Mirror/Direct）
   - 生成优化的图片提示词

2. **fal.ai Grok Imagine Edit API**：
   - 基于参考图片生成自拍
   - 保持人物特征一致性
   - 支持场景化编辑

3. **LLM 对话引擎**：
   - 支持自定义性格特征
   - 对话上下文管理
   - 智能判断何时生成自拍

### 成本优化建议

1. **减少图片生成频率**：不是每次对话都生成自拍
2. **使用更便宜的 LLM**：如 Grok 2 Fast（比 GPT-4 便宜 10 倍）
3. **批量生成**：一次对话生成多张自拍，减少 API 调用次数

## 常见问题

### 1. 为什么生成的自拍和参考照片不像？

**原因**：参考照片质量不够好，或者提示词描述不准确。

**解决方案**：
- 上传高质量的正面照片（512x512 以上）
- 在"外貌描述"中详细描述特征
- 尝试重新生成

### 2. 为什么 AI 不生成自拍？

**原因**：对话中没有触发自拍关键词。

**解决方案**：
- 明确提到"照片"、"自拍"等关键词
- 询问"你在哪"、"你在干嘛"等问题
- 提到"穿"、"看看"等词汇

### 3. 生成的自拍场景不对怎么办？

**原因**：提示词模式选择不当。

**解决方案**：
- 更明确地描述场景（例如："在咖啡厅给我发张自拍"）
- 使用更具体的关键词（例如："穿着红裙子在镜子前自拍"）

### 4. 对话不够自然怎么办？

**原因**：性格描述不够详细。

**解决方案**：
- 在设置中更新女友配置
- 详细描述性格特征、说话风格
- 添加更多兴趣爱好信息

### 5. 成本太高怎么办？

**解决方案**：
- 减少自拍生成频率
- 使用内置 LLM API（不配置 LLM API Key）
- 使用更便宜的 LLM 模型

## 隐私和安全

1. **数据存储**：所有数据存储在你的私有数据库中
2. **图片存储**：参考照片和生成的自拍存储在 S3 中
3. **API Keys**：API Keys 加密存储，不会泄露
4. **对话记录**：只有你可以访问自己的对话记录
